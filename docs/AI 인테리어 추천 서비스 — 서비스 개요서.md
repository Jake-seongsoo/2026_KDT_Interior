## PROBLEM · 문제 정의 (Pain Point)

> 인테리어 정보는 넘쳐나지만, 오늘의집·유튜브·블로그의 모든 정보는 **'다른 사람의 공간'**을 기준으로 만들어진다.
> 내 도면·내 예산·내 취향의 교집합을 찾아주는 것이 없다.

### PROBLEM 01 · 선택지 과잉 → 결정 피로

가구·가전·자재의 선택지가 너무 많아 비교 기준을 스스로 세울 수 없다. FOMO와 후회 비용 때문에 결정 자체를 미루게 된다.

### PROBLEM 02 · 내 공간 시각화 불가

오늘의집·인스타 사진이 예뻐도 "내 집에 어떻게 적용할지"를 알 수 없다. 설치 후에야 결과를 볼 수 있는 미리보기의 부재.

### PROBLEM 03 · 경험 축적 불가 구조

인테리어는 일생에 1~2번뿐이라 배울 기회가 없다. "반평생 전세만 살아봐서 처음 해보는 건데"처럼, 시스템 자체가 경험 축적을 허용하지 않는 구조적 장벽이다.

### PROBLEM 04 · 비가역 고비용 + 예산 불확실성

수백~수천만 원, 한번 설치하면 바꿀 수 없다. 업체마다 다른 견적 기준, breakdown 없는 총액 견적으로 '정확한 예산'조차 파악이 불가능하다. 정보 비대칭 속 결정 강요 → 통제감 상실.

### PROBLEM 05 · SNS 기대치 → 내 집과의 괴리감

오늘의집·인스타의 '완성된 집' 이미지가 기준점이 된다. 하지만 그 이미지는 전문 스타일링·특정 조명·과장된 연출의 결과다. 똑같이 따라 해도 "왜 내 집은 이렇게 차이가 나지?" — SNS가 만든 비현실적 기대치가 결정을 더 어렵게 만들고, 구매 후 박탈감으로 이어진다.

## WORKAROUNDS · 현재 사용자들의 대처 방법

> 사용자들은 이미 5가지 문제를 풀기 위해 **핀터레스트·엑셀·손그림·카카오톡** 등 비공식 방법을 동원하고 있다.
> 그러나 모든 방법이 같은 결론에 도달한다 — **파편화·맥락 불일치·정보 휘발**. 시작은 있지만 결정으로 이어지지 않는다.

### P01 · 선택지 과잉 대처

| 방법 | 한계 |
|------|------|
| 📌 핀터레스트 무드보드 (수백 장 저장) | "300장 저장했는데 막상 무슨 스타일인지 모르겠다" |
| 💬 카카오톡 단톡 투표 | 비동기 소통 → 결정 속도만 늦춤 |
| 🙋 커뮤니티 "봐주세요" 게시글 | 댓글 20개가 10:10으로 갈려 더 혼란 |

**⚠ 핵심 한계:** 스타일 수렴이 안 되고, 비동기 의견 수렴은 결정 속도만 늦춤

### P02 · 시각화 불가 대처

| 방법 | 한계 |
|------|------|
| ✏️ 줄자 + 모눈종이 손그림 평면도 | "문 열리는 방향 깜빡해서 소파 놓고 문이 안 열림" |
| 🪑 이케아 연필·방안지 배치 시뮬레이션 | 2D 한계, 문·콘센트 위치 누락 잦음 |
| 📱 Magic Plan, Planner 5D 등 무료 앱 | 국내 가구(한샘·일룸) DB 없음, 학습 곡선 높음 |

**⚠ 핵심 한계:** 2D 한계 + 국내 가구 DB 부재

### P03 · 경험 축적 불가 대처

| 방법 | 한계 |
|------|------|
| 📖 블로그 200개 독학 | "내 상황에 딱 맞는 글이 없더라" |
| ▶ 유튜브 인테리어 채널 시청 | 시청 ≠ 적용 능력, 정보 미축적 |
| 🤝 지인 찬스 | "그 업체 우리 동네 안 온대요" — 맥락 불일치 |

**⚠ 핵심 한계:** 정보는 쌓이지만 의사결정으로 연결 안 됨

### P04 · 예산 불확실성 대처

| 방법 | 한계 |
|------|------|
| 📊 엑셀 견적 비교표 직접 작성 | "항목 이름이 다 달라서 비교 자체가 불가능했음" |
| 🏠 입주민 밴드·오픈카톡 정보 공유 | 정보 채팅에 묻혀 검색 불가, 이후 휘발 |
| 🔢 "평당 단가" 블로그 역산 | 광고성 콘텐츠 多, 포함 범위마다 의미 다름 |

**⚠ 핵심 한계:** 견적 비표준화 + 적정가 판단 기준 없음

### P05 · SNS 갭 대처

| 방법 | 한계 |
|------|------|
| #️⃣ #저예산 해시태그 필터링 | 노동력·지역 변수 미표시 |
| 🖼 Before·After 사진 수집 | Before 집도 내 집보다 좋은 상태인 경우 많음 |
| 🔆 조명·커튼만 교체로 타협 | 타협이지 해결이 아님 |
| 📽 광각 렌즈 효과 학습으로 원인 이해 | 이해해도 만족은 안 됨 |

**⚠ 핵심 한계:** SNS 구조상 필터링 불가. 결국 "완벽한 인테리어 포기 → 소품 교체"로 타협

> *"오늘의집은 비현실적인 거 알면서도 계속 보게 됨. 그러다 내 집 보면 우울해짐"*

### 횡단 패턴 — 모든 문제를 관통하는 3가지 행동

| # | 패턴 | 내용 |
|---|------|------|
| INSIGHT 01 | **결정의 사회화** | 혼자 결정하지 못하고 커뮤니티·가족·지인에게 판단을 위임한다. "틀리지 않았다"는 검증 욕구가 핵심 — AI 근거 추천이 이를 대체할 수 있다. |
| INSIGHT 02 | **정보 과잉 → 지인 추천 회귀** | 블로그·유튜브·카페를 다 거쳐도 결국 지인 추천 업체로 계약한다. 정보 탐색이 의사결정으로 연결되지 않는 구조적 단절이 존재한다. |
| INSIGHT 03 | **이케아 = 유일한 프로세스** | 이케아 연필·방안지·치수표가 "어떻게 시작해야 하는가"를 알려주는 유일한 오프라인 채널로 인식된다. 사용자는 제품이 아닌 접근법이 필요하다. |

## SOLUTION · 해결 방법 — AI 처리 파이프라인

```
🏠 STEP 01       🔍 STEP 02       🌐 STEP 03                    ✨ STEP 04          🛒 STEP 05
도면 업로드  →  Vision 분석  →  ┌ [AI 자동] 톤 6개 추천 ──────┐
JPG/PNG          방 이름·개수     │  도면 기반 다양화              │  톤 선택  →  방별 시안 + 상품
최대 5 MB        자동 추출       └ [직접 입력] 톤 3개 변형 ─────┘  선택 1회     방별 이미지 1장 + 이케아 상품
[User Input]   [Claude Vision]     안전·중립·대담         [User Select] [Imagen 4 + IKEA]
                                  자유 텍스트 + 무드 칩
```

> **🌟 신규 — SNS 영감 번역 모드 "내 도면으로 번역하기"**
>
> `도면 + SNS 영감 이미지 → Vision×2(공간 분석 + 스타일 추출 + 벽 분류) → 호환성 매핑 → 가능·불가·대안 분류 + Imagen 렌더링`
>
> 영감 이미지를 첨부하면 Web Search 트렌드 탐색 단계가 **"영감 분석 + 내 도면 호환성 매핑"** 으로 대체됩니다.
> 벽 두께 기반으로 **확장 가능 영역(칸막이벽)** 과 **불가 영역(내력벽)** 을 자동 분류합니다.

## BENEFITS · 고객 혜택 및 기대 효과

### ⏱️ 의사결정 시간 단축

도면 업로드 후 방 구성을 분석하고, 도면 특성에 맞는 **6개 톤**을 먼저 비교합니다. 사용자가 톤을 하나 고르면 주요 방별 시안이 생성되어, 수일~수주 걸리던 스타일 탐색 시간을 대폭 줄입니다.

### 💰 예산 불확실성 해소

공정별 표준 단가 기반으로 예산 범위를 사전에 파악합니다. 업체마다 다른 견적 기준에 흔들리지 않고, **"이 금액이 적정한가"를 근거 있게 판단**할 수 있습니다.

### 😌 인테리어 스트레스 감소

"이게 맞는 선택일까?"라는 불안을 AI가 제안한 6개 톤과 방별 시안이 줄여줍니다. 배우자·가족·친구에게 공유 링크를 보내 함께 검토할 수 있습니다.

### 🏡 내 공간에 대한 통제감 회복

"내 도면 기반" 추천이라는 사실 하나가 결정의 주도권을 돌려줍니다. 전문가 없이도 **"이 선택이 맞다"는 근거**를 손에 쥐고 결정할 수 있습니다.

### 📱 SNS 기대치와 내 집 현실의 간극 해소

**"내 도면으로 번역하기"** 기능으로 오늘의집·인스타 영감 이미지를 가능·불가·대안으로 자동 분류하고, Gemini Imagen으로 내 집 버전을 미리 확인합니다. 벽 구조 분석으로 확장 가능 영역까지 파악해 **"포기 → 조명·커튼만 교체"** 타협 경로를 끊고, SNS 영감을 내 공간 조건에 맞게 현실화합니다.
