## Phase 1 — 핵심 AI 파이프라인 (4~5주, 필수)

> 목표: 업로드 → 분석 → 톤 선택 → SVG + 방별 이미지 + 추천 근거가 실제로 동작하는 엔드투엔드 플로우 완성

### 백엔드

| 기능 ID | 기능명 | 세부 내용 |
|---------|--------|----------|
| F001 | 도면 업로드 | multipart/form-data, 5MB 제한, GCS 저장 + Signed URL |
| F002 | Claude Vision 방 분석 | 방 이름·개수·우선순위 추출, POST /analyze |
| F004 | AI 자동 톤 6개 생성 | Claude Web Search 트렌드 캐시(24h TTLCache) + 공간 특성 기반 6개 생성. POST /analyze |
| F004-C | 사용자 입력 기반 톤 3개 변형 | "직접 입력" 모드: 자유 텍스트 + 무드 칩 12개(+ 자유 태그) → Claude 안전·중립·대담 3개 변형. POST /analyze/custom. 트렌드 캐시는 F004와 공유 |
| F004-B | SVG 2D 탑뷰 배치도 생성 | svgwrite, 방 이름·비율 표시, POST /render 내 동기 처리 |
| F004-B | Imagen 방별 렌더링 | Vertex AI Imagen 4, 최대 4개 방 asyncio.gather 병렬, return_exceptions=True |
| F005 | 이케아 상품 검색 | 이케아 비공식 API, 슬롯당 5개, Vision 재랭킹 적용, 병렬 호출 |
| RISK-01 | gather fallback 처리 | Imagen 부분 실패 시 나머지 반환 보장 |
| RISK-02 | 도면 이미지 보안 | GCS Signed URL(15분 TTL), public URL 반환 금지 |
| RISK-03 | JWT 인증 | /analyze, /render, /share 모두 Depends(verify_jwt) 적용 |

### 프론트엔드

| 페이지 | 구현 내용 |
|--------|----------|

| AI 분석 대기 페이지 | 단계별 진행 상태 표시 (분석 중 → 톤 후보 생성 중), **모드별 스텝 텍스트 분기** |

## 개발 시작 권장 순서

```
1. 백엔드 뼈대
   main.py + CORS + 환경변수(config.py) + JWT 미들웨어(auth.py)

2. POST /analyze
   GCS 업로드 → Claude Vision → 톤 후보 6개 생성 → 응답

3. POST /render
   SVG 생성 → Imagen 병렬 렌더링 → 이케아 상품 → DB 저장 → 응답

4. 프론트엔드 핵심 플로우
   업로드 페이지 → 대기 페이지 → 톤 선택 페이지 → 결과 페이지

5. Supabase Auth (Google OAuth)

6. Phase 2 기능 순차 추가
```

## 완료 기준 (Definition of Done)

- **Phase 1 완료** = 실제 아파트 도면 JPG를 업로드하면 방별 Imagen 이미지와 SVG 배치도, 추천 근거 텍스트가 결과 페이지에 표시된다. AI 자동(톤 6개)·직접 입력(톤 3개) 두 모드 모두 동작한다
- **Phase 2 완료** = 톤 재선택이 동작하고, 방 면적이 UI에 표시되며, AI 생성 이미지 고지가 표시된다
- **Phase 3 완료** = 정밀화 모달(신혼 가전 배치 포함)이 완성되고, 랜딩 페이지가 polish되어 포트폴리오로 제출 가능한 상태다
