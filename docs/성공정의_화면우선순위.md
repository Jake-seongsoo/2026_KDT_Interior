# Moodie · 성공 정의와 화면 우선순위

> 작성일: 2026-06-17 | KDT AI/LLM 과정 포트폴리오
> **이 문서의 역할:** 서비스 개요의 목적("혼자서도 확신 있게 결정을 끝낸다")과 사용자의 가장 깊은 동기("틀리지 않았다는 검증")를 한 점으로 모아, 이 과업의 성공을 **관찰 가능한 한 행동**으로 정의한다. 그 행동이 화면에서 가장 분명해지도록 무엇을 키우고 무엇을 뒤로 물릴지 정리한다.
> 근거 체인: `페르소나_정의.md` → `JTBD.md` → `사용자_여정_AsIs.md` → `감정_지도.md` → `UX_표준_벤치마크.md` → **이 문서**.

---

## 1. 성공 = 관찰 가능한 한 행동

> **"사용자가 하나의 결과를 골라 **공유한다**(공유 링크 복사 → 배우자·지인에게 전송)."**

### 왜 이 행동인가

다른 후보(결과 도달, 상품 클릭, PDF 저장)보다 **이 한 행동이 목적·동기를 동시에 증명**한다.

| 이 행동이 증명하는 것 | 근거 |
|----------------------|------|
| **결정을 닫았다 (decision closure)** | "이게 내 답"이라고 굳혀야 남에게 보낸다 — JTBD의 ✅ "이 방향이면 됐다" 신호 |
| **검증 동기를 충족했다 (결정의 사회화)** | 사용자의 1순위 동기 = "혼자 잘못 정한 게 아니다" 확인 — As-Is 6단계 단톡 검증을 Moodie 안에서 끝냄 |
| **포트폴리오 목적도 달성** | 공유 = 지인 베타 확산(개요서 F008 의도) |

### 관측 방법 / 지표

- **관측:** `handleShare` 클릭 이벤트 (`frontend/app/result/[id]/page.tsx`).
- **성공 지표(한 줄):** *결과 페이지 도달자 중 1회 이상 공유한 비율.* (저장·기록은 보조 신호)
- 결과 도달이나 상품 클릭은 **수동적 퍼널 통과**일 뿐 "확신"을 증명하지 못한다. 공유는 사용자가 **자기 이름을 걸고 결과를 외부화**하는, 확신의 가장 명확한 행동이다.

---

## 2. 이 행동이 화면에서 가장 분명해지려면

원칙: **"확신을 키우고 → 공유를 정점으로 올리고 → 결정을 흐리는 경쟁 동선을 물린다."**

### 🔼 키울 것 (Amplify)

1. **공유를 결과 화면의 1순위(primary) CTA로 격상.**
   지금은 다크 히어로에 `홈으로·다른 톤·정밀화·공유·PDF` **5개가 같은 outline 비중**으로 나열돼 공유가 묻힘 (`result/[id]/page.tsx`의 액션 버튼 영역). 공유만 채워진 primary 버튼 + 동기형 라벨("**배우자에게 보내기**")로 시선·터치를 모은다.

2. **선택한 톤의 "확신 근거"를 키운다.**
   공유는 *확신이 선 다음에야* 일어난다. 톤 카드의 설명·추천 이유가 `line-clamp`로 잘려 근거가 안 보임 (`ToneCandidateGrid.tsx`) → 결과 히어로에서 "왜 이 톤인지"를 한 문단 펼쳐 보여 "이게 맞다"를 굳힌다(미스핏 A·B4와 동일).

3. **"내 답" 요약을 한눈에.**
   선택 톤 + 방별 대표 1장 + 팔레트를 상단에 압축해, *공유 직전에 "이거면 됐다"* 가 시각적으로 완성되게 한다(공유 미리보기/OG도 자랑스럽게).

### 🔽 뒤로 물릴 것 (De-emphasize)

1. **재탐색 동선(`다른 톤`·`정밀화`)을 secondary로.**
   이 둘은 *"아직 확신 못 함"* 신호 — 결정 피로 사용자에게 결정을 **다시 미루게** 만든다. 시각 비중을 낮추고("마음에 안 들면…" 보조 영역으로), 결과에 만족한 사용자의 눈에는 공유가 먼저 들어오게 한다.

2. **`PDF 저장`·`홈으로`는 부가 동선으로.**
   PDF는 모바일에서 약하고(미스핏 D) 공유와 경쟁. 아이콘/텍스트 비중을 낮춰 1순위 자리를 공유에 양보한다.

3. **상품 그리드의 화면 점유를 결정 이후로.**
   구매는 *결정을 닫은 다음* 행동. 공유 결정 전에 상품이 스크롤을 길게 점유하면 "고르는 일"이 다시 시작되는 느낌을 준다 → 방별 상품은 탭 하단/접힘 우선으로 물려, **확신 → 공유** 흐름을 끊지 않게 한다.

---

## 3. 한 줄 원칙과 주의

- **원칙:** 결과 화면은 "더 탐색하라"가 아니라 **"이게 네 답이다, 보여줘라"** 가 가장 크게 들리는 화면이어야 한다.
- **주의(회귀 위험):** `다른 톤`/`정밀화`를 *제거*하면 안 됨(F006·F007 핵심 가치). **숨기는 게 아니라 비중을 낮추는** 재배치이며, 관련 E2E(버튼 존재·라우팅)는 `data-testid` 기준으로 유지·갱신 필요.
